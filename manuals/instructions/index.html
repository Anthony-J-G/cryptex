Introduction

Welcome to the Cryptex! If you’ve found yourself here then you more than likely are here to practice your programming, computer science, and some mathematics in a game-like setting. If you aren’t, well then I’m really confused how you got here, but the more the merrier!

The cryptex is heavily inspired by classic choose your own adventure novels, tabletop RPGs (such as Dungeons & Dragons), as well as multi-user dungeons (sometimes referred to as MUDs). This tutorial will provide a quick crash course on the basics of programming. It uses a gaming scenerio to contextualize the teachings, and requires some self-motivation in learning.

Some sources you can reference for help understanding the fundamentals are: Princeton University’s Overview of “Basic Programming Concpets”, your lanugage’s respective manual/documentation (ex. Python’s official documentation), StackOverflow which is an active hub of people asking and answering tech questions.

Dependencies

There are a few dependencies that are needed before we get started. Some of these are required, some will be optional quality of life features whose use will become evident later on.

Programming Languages

The user will interact with the game via the Python programming language and their system’s terminal. However, the concepts that we’ll be using can be more broadly applied to any programming language of the user’s choice that allows them to do basic HTTP requests and cryptography. If you don’t know what either of those things are, that’s ok we’ll touch on them later on.

Python is the default language of choice due to it’s relative simplicity, readability, and easy access/integration of 3rd party libraries. For more information on how to install Python, visit the official Python website.

Additionally, two of Python’s strongest features are the virtual enviroments and pip package manager. The tutorial will assume that you are using these features. The virtual enviornments help keep 3rd party content in one project from muddying the rest of your Python projects and pip provides a simple interface for handling package fetching and versioning for you.

Source Control

Git has pretty quickly become ubiquitous and the de facto default version control system. A large motivation for using this is ensuring that you can edit files freely without needing to worry about losing your progress. If you are new to Git, it definitely has a steep learning curve, but using it is very approachable in bite sized chunks (there’s a pun here and you know it). It’s also a good practice to get into the habit of using some kind of version control for any software project. For an overview of the basics checkout either of these resources, Git’s official quickstart guide and GitHub’s official documentation

Text Editor

This tutorial assumes that you are using VSCodium for editing files and executing code. VSCodium is a fork of Microsoft’s Visual Studio Code that contains only software that follows “Free and Open Source Software” (FOSS) principles. As a note and paraphrasing Richard Stallman, the “free” here means “free” as in “free speech”, not “free” as in “free sandwich” (though VSCodium follows both defintions).

However, you can use any program that allows you to edit files. I got started with programming using Windows’ builtin Notepad text editor and a terminal instance, if you want to use that then I wish you the best of luck. Having a good editor for files in modern programming can be crucial in development. However this game assumes we will program like we are living in 2025 instead of 2005; there are a plethora of options available.

Microsoft’s Visual Studio Code has grown with popularity over time, as well as becoming a much more mature open source file editor. It supports many helpful extensions for many different languages and has a lot of support for quality of life features. This is not to say that it doesn’t have it’s downsides. It’s occasionally slowed down by Microsoft’s background telemetry processes and its close integration with AI (along with other AI code editors derived from it) do not fit the learning experience that this game intends to cultivate.

Some alternatives to any of the Visual Studio Code derived editors are NeoVim, which is a very high powered editor with an equally high learning curve, and classic Visual Studio, which has a lot of unnecessary builtin features not needed with Python along with more of Microsoft’s strings attached. JetBrains also has a choice of editors that are more than comparable, as well as Sublime Text.

Setting up the Enviroment

In order to complete our setup of both Git and Python, we need to create the Git repository, create/activate the Python virtual environment, and add some configuration files.

To start, open up a new terminal from inside VSCodium by clicking the “Terminal” option in the top menu and selecting “New Terminal”. The “Terminal” option might be hidden depending on your editor’s default configuration, but clicking the ellipsis icon in the top menu should reveal it.
